Uppgift 4.1

För att matcha liu-id använde följande:

^[a-z]{4,5}\d{2,3}$

^ är start of line, [a-z]{4,5} matchar 4 eller 5 bokstäver, \d{2,3} matchar 2 eller 3 siffror och $ är end of line


Uppgift 4.2

För att matcha ett ISO-datum (YYYY-MM-DD) använde följande:

^(\d{4}-\d{2}-\d{2})$


Uppgift 4.3

För att kontrollera rimlighet använde följande:

^\d{4}-(0[1-9]|1[0-2])-(0[1-9]|[12]\d|3[01])$

^ är start of line, \d{4} matchar årets 4 siffror (0000-9999), (0[1-9]|1[0-2]) matchar månaden 01-09 eller 10-12, (0[1-9]|[12]\d|3[01]) matchar dagen 01-09, 10-29, 30-31


Uppgift 4.4

För att matcha alla taggar använde följande:

import re 
file_path = input("path to file: ")
with open(file_path, "r") as file:
    html = file.read()
    tags = re.findall(r"\s*<(\w+)[^>]*>", html)
    print(tags)

\s* matchar 0 eller flera whitespace-tecken som möjligtvis föregår taggen om man har indentering, < är start of tag, ([A-Za-z]+) matchar 1 eller flera bokstäver - själva taggen och capture den, [^>]* matchar 0 eller flera tecken som inte är > och därmed ignorerar allt efter taggen, (för att ignorera alla attribut och liknande) och > är end of tag


Uppgift 4.5

GET används för att hämta en data, och kan innehålla GET-parametrar i URL:en i form av ?key1=value1&key2=value2. Då kan den här requesten bokmarkas (sparas för senare). GET-parametrar ska inte innehålla känsliga data som lösenord eller personlig information.

POST används för att skicka data till server och innehåller data i body.

GET används när man vill läsa/hämta information. POST används när man skickar in information, särskilt om den är större i mängd.

På min blogg används GET för att hämta statisk html-fil. Detta kan man kontrollera genom att gå in på DevTools > Network tab, uppdatera sidan och kolla under Method kolumn

Uppgift 2.6 - Smått och gott

find är ett kommando som söker efter filer i kataloger, -type f söker efter filer, -executable söker efter executable filer och ~ står för home katalog

find ~ -type f -executable


-name "*.py" söker efter filer som har extensionen .py och -exec grep -l '#!/usr/bin/env python3' {} \; söker efter filer som innehåller #!/usr/bin/env python3. grep -l outputar filnamnen som matchar sökningen och {} \; kör grep på varje fil som hittas av föregående find constraints

find ~ -type f -executable -name "*.py" -exec grep -l '#!/usr/bin/env python3' {} \;


-maxdepth 1 söker efter filer i kataloger på djupet 1 dvs endast i nuvarande katalog

find ./ -maxdepth 1 -type f -exec grep '^#' {} \;

Sedan skapade jag en funktion som kontrollerar om ett tal är ett primtal.
Först hanterade jag edgecases som talet är mindre än 2 eller delbart med 2, då är det inte ett primtal. Sedan kontrollerade jag om talet är 2, då är det ett primtal.
Efter det räknade jag ut roten av talet (största möjliga delare) och kontrollerade om talet är delbart med något ojämnt tal mellan 3 och roten av talet, då är det inte ett primtal. 

function isPrime {
    n=$1
    if [ "$n" -lt 2 ]; then
        echo "false"
        return
    fi
    if [ $(($n % 2)) -eq 0 ]; then
        echo "false"
        return
    fi
    if [ "$n" -eq 2 ]; then
        echo "true"
        return
    fi

    local upperLimit=$(echo "scale=0; sqrt($n)" | bc -l)
    local COUNTER=3

    while [[ $COUNTER -le $upperLimit ]]; do
        if [ $(($n % $COUNTER)) -eq 0 ]; then
            echo "false"
            return
        fi
        let COUNTER=$(($COUNTER+2))
    done
    echo "true"
}
